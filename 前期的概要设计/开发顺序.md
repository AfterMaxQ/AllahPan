
## 框架构思的9个步骤

### 步骤1：数据持久层（根基）
- **SQLite**：用户、文件元数据
- **ChromaDB**：AI语义向量
- **Mac本地文件系统**：实际文件存储
- 先确定数据模型，因为上层都依赖它

### 步骤2：AI推理层（核心能力）
- **Ollama引擎**：本地推理服务
- **Qwen3-VL-4B**：图片解析模型
- **文本向量化**：生成语义向量
- 这是项目的差异化竞争力

### 步骤3：目录监听与同步服务（特色功能）
- **watchdog/fsevents**：监控文件系统变化
- **同步逻辑**：文件变化→更新数据库→清理索引
- 实现"无系统删除"的核心约束

### 步骤4：应用服务层（业务逻辑）
- **用户管理**：注册、登录、JWT认证
- **文件服务**：上传、下载、预览
- **AI语义服务**：搜索、解析
- **系统服务**：监控、日志

### 步骤5：API接口层（对外契约）
- RESTful API设计
- 统一响应格式
- 认证中间件
- Swagger文档

### 步骤6：前端层（双前端体系）
- **PySide6桌面端**：完整功能、原生体验
- **移动端Web**：轻量化、自适应

### 步骤7：网络层（本地+远程）
- **本地模式**：127.0.0.1访问
- **远程模式**：Cloudflare Tunnel + 阿里云域名

### 步骤8：一体化打包层（交付形态）
- **启动器**：统一调度所有服务
- **打包结构**：macOS .app格式
- **启停流程**：对标SpringBoot体验

### 步骤9：目录结构整合
- 按功能模块组织代码
- 清晰的分层边界
- 便于开发和维护

## 核心设计原则

| 原则 | 说明 |
|------|------|
| **分层清晰** | 每层职责单一，接口明确 |
| **依赖向下** | 上层依赖下层，下层独立 |
| **模块化** | 各模块可独立开发测试 |
| **可扩展** | 新增功能不破坏现有结构 |

## 开发顺序建议

```
数据层 → AI层 → 目录监听 → 服务层 → API层 → 后端测试 → 前端开发 → 打包集成
```

这个框架设计的核心思想是：**从底层基础设施开始，逐步向上构建，每一层都为上层提供稳定的服务契约**。这样既能保证系统的稳定性，又能让各层并行开发。
