# 🚀 AllahPan 极简 Web 前端 - 快速启动指南

## 📌 如何使用 Web 前端（两步）

1. **启动后端**（在项目目录下）  
   ```bash
   cd backend
   python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
   ```

2. **打开前端页面**（任选其一）  
   - **方式 A**：双击 `frontend_web/index.html` 用浏览器打开（API 默认指向当前页面的 origin；若用 file:// 打开，会回退到 `http://localhost:8000/api/v1`）。  
   - **方式 B（推荐）**：在 `frontend_web` 下运行 `python run.py`，浏览器会自动打开 `http://localhost:3000`。

接着在页面中**注册/登录**，即可使用：上传/下载文件、目录浏览、AI 语义搜索。  
**外网访问**（内网穿透）请直接看下方「远程访问与内网穿透」一节。

## 📝 API 测试（使用 curl 或 Postman）

### 注册
```bash
curl -X POST http://localhost:8000/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123","email":"admin@example.com"}'
```

### 登录
```bash
curl -X POST http://localhost:8000/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123"}'
```

### 获取文件列表
```bash
curl -X GET http://localhost:8000/api/v1/files/list \
  -H "Authorization: Bearer YOUR_TOKEN"
```

### 上传文件
```bash
curl -X POST http://localhost:8000/api/v1/files/upload \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "file=@/path/to/your/file.jpg"
```

### AI 搜索
```bash
curl -X POST http://localhost:8000/api/v1/ai/search \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"query":"风景照片","limit":10}'
```

## 🔧 已修复的问题

### 问题 1：bcrypt 密码加密兼容性
**原因**：passlib 与新版 bcrypt 存在兼容性问题

**解决方案**：直接使用 bcrypt 库进行密码加密
```python
# 加密
salt = bcrypt.gensalt()
hashed = bcrypt.hashpw(password.encode('utf-8'), salt)

# 验证
bcrypt.checkpw(plain_password.encode('utf-8'), hashed_password.encode('utf-8'))
```

## 🌐 远程访问与内网穿透（外网访问家庭主机）

AllahPan 已集成 **Cloudflare Tunnel**，可在不开放家庭路由器端口的情况下，通过公网 HTTPS 访问家里的后端服务。

### 方式一：局域网访问（同一 WiFi）

1. 后端启动时使用 `--host 0.0.0.0`，确保监听所有网卡。
2. 在手机/其他电脑浏览器输入：`http://家庭主机IP:8000`（需先知道主机在局域网中的 IP）。
3. 若前端单独用 HTTP 服务器打开（如 `run.py` 在 3000 端口），前端需指向后端：在打开 `index.html` 的页面前执行：
   ```html
   <script>window.ALLAH_PAN_API_BASE = 'http://家庭主机IP:8000/api/v1';</script>
   ```
   或直接访问 `http://家庭主机IP:3000`（前端跑在 3000 时）。

### 方式二：Cloudflare Tunnel 内网穿透（外网任意地点访问）

**前提**：家庭主机上已安装并配置好 Cloudflare Tunnel（见下方步骤）。

#### 1. 安装 cloudflared

- **Windows**：从 [Cloudflare 文档](https://developers.cloudflare.com/cloudflare-one/connections/connect-apps/install-and-setup/installation/) 下载 `cloudflared`，或使用 `winget install cloudflared`。
- **macOS**：`brew install cloudflared`。
- **Linux**：按官方文档安装，或使用包管理器。

#### 2. 在 Cloudflare 创建隧道并获取 Token

1. 登录 [Cloudflare Zero Trust 控制台](https://one.dash.cloudflare.com/)（或 Cloudflare 仪表板 → Zero Trust）。
2. 进入 **Networks** → **Tunnels**，点击 **Create a tunnel**。
3. 选择 **Cloudflared**，命名（如 `allahpan-home`），下一步。
4. 在 **Public Hostname** 中：
   - Subdomain 填你想要的子域名（如 `allahpan`），或使用 Cloudflare 提供的随机域名。
   - 服务类型选 **HTTP**，URL 填 **localhost:8000**（即本机 AllahPan 后端端口）。
5. 创建完成后，在 **Install and run a connector** 页面复制 **Tunnel Token**（一串长字符）。  
   **注意**：Token 只显示一次，请妥善保存。

#### 3. 在 AllahPan 中配置并启动隧道

- **使用桌面端（推荐）**  
  1. 打开 AllahPan 桌面客户端。  
  2. 进入 **设置** → **远程访问** → **配置远程访问...**  
  3. 粘贴刚才复制的 **Tunnel Token**，可选填自定义域名（若在 Cloudflare 已配置）。  
  4. 保存后，在设置页或状态栏点击 **启动远程访问**。  
  5. 状态显示「已连接」后，会看到公网访问地址（如 `https://xxx.your-domain.com`）。

- **仅使用 Web 前端时**（后端在同一台机器上）：  
  隧道需由本机后端管理。可用桌面端配置并启动一次；或直接调用后端 API 配置并启动（需先登录获取 JWT）：
  - 配置：`POST /api/v1/tunnel/config`，body：`{ "token": "你的Tunnel Token" }`
  - 启动：`POST /api/v1/tunnel/start`

#### 4. 外网使用网页端

隧道暴露的是 **后端**（端口 8000），公网地址为：`https://你的隧道域名`，API 基础地址为：`https://你的隧道域名/api/v1`。

- **在任意设备浏览器中**：  
  1. 先在一个能打开「网页前端」的页面里指定 API 地址。例如把 `frontend_web/index.html` 拷到 U 盘/网盘，或部署到任意静态托管。  
  2. 在该页面**最先**执行（或写一个小的引导页）：
     ```html
     <script>window.ALLAH_PAN_API_BASE = 'https://你的隧道域名/api/v1';</script>
     ```
  3. 然后打开或加载 `index.html`（确保在 `api.js` 之前执行上述脚本）。  
  4. 访问地址即为「你的隧道域名」对应的 HTTPS 地址，登录后即可使用文件、搜索等功能。

- **若希望「只打开一个网址」就进网页端**：  
  可将 `frontend_web` 作为静态资源由后端托管（需在后端挂载静态目录并设置默认页为 `index.html`），这样访问 `https://你的隧道域名/` 即为网页端，无需单独设 `ALLAH_PAN_API_BASE`（与后端同源即可）。

### 安全提示

- Tunnel Token 相当于隧道密钥，不要泄露或提交到代码库。
- 外网访问依赖后端 JWT 登录，请使用强密码并定期更换。
- 仅在可信任设备上使用「记住登录」；公共设备上使用后建议退出登录。
