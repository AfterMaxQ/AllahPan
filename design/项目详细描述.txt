# Allah家庭私有网盘 详细设计文档

**核心约束**：无系统级文件删除功能，用户通过Mac本地目录手动删除；固定AI模型Qwen3-VL-4B；SQLite+ChromaDB双存储；双前端体系（PySide6桌面端+移动端Web端）；前后端一体化打包为macOS原生应用；AI推理框架采用Ollama；支持Cloudflare Tunnel+阿里云域名远程访问；全本地化部署，适配Mac Studio M2 Max；运行时Python 3.12

---

## 二、系统总体详细架构
### 2.1 架构分层
系统采用**七层一体化闭环架构**，支持本地/远程双访问模式，所有模块解耦、可独立维护，最终打包为单应用：
1. **访问层**：本地桌面端 + 远程Web端 + 移动浏览器端
2. **网络层**：本地局域网 + Cloudflare Tunnel加密远程隧道（阿里云域名）
3. **前端层**：PySide6桌面客户端、自适应移动端Web页面
4. **应用服务层**：FastAPI后端核心服务集群
5. **AI推理层**：Ollama推理引擎 + Qwen3-VL-4B多模态模型
6. **数据持久层**：SQLite元数据、ChromaDB向量库、Mac本地文件系统
7. **一体化打包层**：macOS .app应用封装，包含所有组件

### 2.2 模块交互逻辑
1. 本地/远程用户通过前端发起操作请求，经网络层传输至后端
2. 后端统一调度业务服务、Ollama AI引擎、数据层完成处理
3. 文件物理存储于Mac本地，用户手动管理删除
4. 目录监听服务自动同步文件状态，清理无效数据索引
5. 远程访问流量经Cloudflare加密隧道转发，无公网端口暴露
6. 所有组件内置打包，启动器统一调度启停

### 2.3 远程访问架构
通过Cloudflare Tunnel建立家庭内网与公网的加密隧道，绑定阿里云购买的自定义域名，实现全球无公网IP远程访问，移动端/异地电脑通过浏览器直接访问网盘服务。

## 三、一体化打包设计（核心新增）
### 3.1 打包形态
- **输出格式**：`Allah.app`（macOS原生应用安装包）
- **体积**：包含AI模型约2~3GB（可裁剪量化模型）
- **运行方式**：**双击即用**，无任何环境配置、无依赖安装
- **分发方式**：单文件拷贝，可备份、可分发至任意Mac设备

### 3.2 内置组件（全部打包进安装包）
1. **前端**：PySide6桌面客户端
2. **后端**：FastAPI + Uvicorn（内置Web服务）
3. **AI引擎**：Ollama推理引擎主程序 + Qwen3-VL-4B量化模型
4. **数据库**：SQLite（单文件）、ChromaDB（向量库）
5. **系统服务**：目录监听、日志、存储监控
6. **可选**：Cloudflare Tunnel远程隧道客户端
7. **运行环境**：内置Python 3.12解释器 + 所有依赖库

### 3.3 启动流程（对标SpringBoot）
1. 双击`Allah.app`
2. 一体化启动器初始化：创建本地数据目录、加载数据库
3. **后台启动Ollama推理引擎**，自动加载Qwen3-VL-4B模型
4. **后台启动FastAPI后端服务**（监听127.0.0.1:8000）
5. **前台打开PySide6前端窗口**
6. 前端自动连接本地后端
7. 可选：自动启动Cloudflare隧道
8. 进入网盘主界面，直接使用

### 3.4 关闭流程
1. 关闭PySide6前端窗口
2. 自动停止FastAPI后端服务
3. **自动关闭Ollama推理引擎**
4. 自动关闭所有子进程
5. 数据自动保存，无残留进程

## 五、应用服务层详细设计（FastAPI）
### 5.1 核心服务模块
#### 5.1.1 用户管理服务
负责本地账号的创建、身份验证、密码加密存储，所有用户共享系统全部文件。

#### 5.1.2 文件服务
提供文件异步上传、分片存储、下载、预览、元数据查询功能，支持全格式文件处理，大文件优化传输效率，所有文件物理存储于Mac本地指定目录。

#### 5.1.3 AI语义服务
通过标准HTTP API调用Ollama推理引擎，完成图片解析、语义理解，调用嵌入模型生成语义向量，对接向量库实现语义检索、多模态搜索，支持上传文件自动生成AI索引。

#### 5.1.4 目录同步服务
通过目录监听机制实时监控Mac本地文件存储目录，用户手动删除文件后，自动清理SQLite中的文件元数据与ChromaDB中的向量索引，保证数据一致性。

#### 5.1.5 系统服务
实现存储空间监控、系统日志记录、后端开机自启、配置管理，适配macOS系统环境，负责Ollama引擎的启停控制。

### 5.2 服务特性
1. 全异步架构，高并发处理文件与搜索请求
2. 无文件删除接口，无回收站相关逻辑
3. 服务间低耦合，可独立启停
4. 支持远程访问统一接口调度
5. 统一管理Ollama AI引擎的生命周期

---

## 六、AI推理层详细设计（Ollama核心）
### 6.1 核心组件
#### 6.1.1 Ollama推理引擎
- **唯一AI推理框架**：Ollama
- **运行环境**：内置打包进一体化应用，静默后台运行
- **核心能力**：模型加载、推理调度、硬件加速、API服务
- **硬件适配**：自动调用Mac Studio M2 Max Neural Engine，推理速度提升3~5倍
- **API接口**：提供本地标准REST API，FastAPI直接调用

#### 6.1.2 Qwen3-VL-4B多模态模型
- **唯一AI模型**：`qwen3-vl:4b`（Ollama官方原生支持）
- **核心能力**：图片OCR文字提取、图片内容语义理解、场景描述
- **应用场景**：图片上传后自动解析文本，为语义搜索提供基础数据
- **部署方式**：内置打包进一体化应用，无需单独下载

#### 6.1.3 文本向量化模型
将Ollama输出的文本内容转换为固定维度语义向量，用于向量检索匹配，模型轻量化，适配本地快速推理。

#### 6.1.4 向量索引管理
基于ChromaDB实现向量的存储、检索、删除，与文件ID绑定，支持全局语义搜索。

### 6.2 AI工作流程
1. 图片文件上传 → 后端向Ollama发送推理请求
2. Ollama调度Qwen3-VL-4B解析生成文本
3. 文本转向量 → 存入向量数据库 → 更新文件is_ai_parsed=True
4. 用户搜索 → 文本转向量 → 向量匹配 → 返回结果
5. 用户手动删除文件 → 目录监听检测 → 自动清理向量索引

## 七、数据持久层详细设计
### 7.1 数据库选型
**SQLite**：单文件嵌入式数据库，无需独立服务，轻量化适配家庭项目，存储用户与文件元数据。
**ChromaDB**：本地向量数据库，存储AI语义向量与解析文本。

### 7.2 数据表结构设计
#### 7.2.1 用户表
- 主键：用户唯一标识（UUID）
- 字段：用户名、加密密码、创建时间
- 约束：用户名唯一，非空校验

#### 7.2.2 文件表
- 主键：文件唯一标识（UUID）
- 字段：所属用户ID、文件名、本地存储路径、文件类型、文件大小、是否已AI解析、上传时间
- 约束：关联用户ID，无删除标记字段
- 说明：用户手动删除文件后，该表对应记录由同步服务自动清理

### 7.3 文件存储规范
1. 所有用户共享同一存储目录
2. 支持自定义根目录，默认路径为Mac文档目录
3. 文件物理删除仅允许用户通过系统访达操作
4. 存储格式保持原文件格式，不做加密修改

## 八、接口详细设计
### 8.1 接口规范

3. **AI接口**：语义搜索、索引重建、图片解析（调用Ollama）
4. **系统接口**：存储信息查询、日志获取、同步状态、Ollama引擎控制

### 8.3 接口约束
- 无文件删除类接口
- 所有接口支持本地/远程双调用
- 接口请求限制：家庭环境低并发优化

## 九、核心业务流程详细设计
### 9.1 文件上传+AI索引流程
1. 用户通过桌面端/Web端上传文件
2. 后端接收并保存文件至本地目录
3. 写入文件元数据至SQLite
4. 图片文件自动向Ollama发送推理请求
5. Ollama调度Qwen3-VL-4B解析内容
6. 解析文本生成语义向量，存入ChromaDB
7. 前端刷新文件列表，完成操作

### 9.2 用户手动删除文件同步流程
1. 用户在Mac访达中手动删除物理文件
2. 目录监听服务检测到文件删除事件
3. 系统根据文件路径匹配SQLite记录并删除
4. 自动删除对应文件的向量库索引
5. 前后端刷新后不再展示已删除文件

### 9.3 AI语义搜索流程
1. 用户输入自然语言搜索内容
2. 后端将搜索文本转换为语义向量
3. 在向量库中检索匹配数据
4. 返回高相关性文件列表
5. 前端展示结果，支持预览

### 9.4 远程访问流程
1. 用户通过浏览器访问阿里云自定义域名
2. 流量经Cloudflare Tunnel加密转发至家庭Mac
3. FastAPI后端接收请求并处理
4. 数据按原路径返回，全程加密传输
5. 移动端自适应展示界面

### 9.5 一体化应用启停流程
1. **启动**：双击Allah.app → 启动器初始化 → 启动Ollama引擎 → 启动FastAPI后端 → 打开PySide6前端
2. **关闭**：关闭PySide6窗口 → 停止FastAPI后端 → 停止Ollama引擎 → 保存数据 → 退出应用

## 十、网络与远程访问详细设计
### 10.1 本地网络模式
- 后端监听局域网地址，家庭内设备可直接访问
- PySide6桌面端连接本地回环地址，低延迟使用
- Ollama引擎仅本地访问，不暴露公网

### 10.2 远程网络模式（Cloudflare Tunnel+阿里云域名）
1. Mac Studio运行Cloudflare隧道客户端（可选内置打包），建立加密连接
2. 阿里云域名解析至Cloudflare节点
3. 公网用户通过域名访问，流量无公网端口暴露
4. 全链路HTTPS加密，支持全球远程访问
5. 移动端/异地设备浏览器直接使用，无需安装APP

### 10.3 安全规范
- 无路由器端口映射，内网完全隐藏
- 域名访问启用Cloudflare防火墙防护
- 用户身份校验，防止非法访问
- Ollama引擎仅本地访问，无公网暴露
